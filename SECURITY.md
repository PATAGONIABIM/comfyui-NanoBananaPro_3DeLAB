# Security Policy / Política de Seguridad

Nos tomamos muy en serio la seguridad de este proyecto. Esta página describe nuestras políticas de soporte y cómo reportar vulnerabilidades.

## Supported Versions (Versiones Soportadas)

Actualmente, solo la versión más reciente (rama `main`) o las versiones mayores actuales de este nodo personalizado reciben actualizaciones de seguridad y mantenimiento continuo.

| Versión     | Soportada          |
| ----------- | ------------------ |
| v1.x (main) | :white_check_mark: |
| < v1.0.0      | :x:                |

*(Te recomendamos mantener siempre actualizado tu repositorio local a la última versión para recibir todos los parches de seguridad).*

## Reporting a Vulnerability (Reportar una Vulnerabilidad)

Si descubres una vulnerabilidad de seguridad en **ComfyUI-NanoBananaPro**, por favor **NO abras un Issue público** en GitHub, ya que esto podría exponer a otros usuarios antes de que podamos solucionarlo.

Por favor, sigue estos pasos:

1. **Contacto Directo:** Envía un mensaje directo o correo electrónico al mantenedor original a: **[contacto@patagoniabim.cl]**.
2. **Qué incluir en el reporte:**
    * Una descripción detallada de la vulnerabilidad (ej. exposición de API keys, inyección de código, etc.).
    * Los pasos exactos para reproducirla (adjuntando un workflow de ComfyUI en JSON si es posible).
    * El impacto potencial del fallo.
    * Tu entorno local (Sistema operativo, versión de Python).

### ¿Qué ocurre después?

* **Respuesta:** Acusaremos recibo de tu reporte lo antes posible (generalmente dentro de 48 horas).
* **Evaluación:** Investigaremos el problema y determinaremos si es necesario publicar un parche urgente. Te mantendremos informado sobre el progreso.
* **Solución y Créditos:** Cuando la solución esté lista y publicada de forma segura, te daremos el crédito correspondiente por tu reporte (si estás de acuerdo) en las notas de la versión.

Agradecemos enormemente a quienes nos ayudan a mantener este ecosistema seguro para todos.
